## Priority Issues

### P0 — Uppercase eyebrow labels (9 instances, absolute ban)
All `uppercase tracking-wide/wider/widest` at `text-[11px]` must be replaced. In a data-dense dark table these labels should guide, not shout. Fix: sentence-case, weight contrast, or a thin left-border rule to group sections without screaming.

### P0 — Contrast failure: inactive toggle buttons
Lines 867, 1142: `text-slate-400` on `bg-blue-600`. The inactive "Perpetuity Growth" / "Exit Multiple" button text (`text-slate-400`) on a blue-600 background fails 4.5:1. Should be `text-blue-100` or `text-white/60`.

### P1 — Composition bar FCF segment invisible
`bg-[#0F2A5E]` (dark navy) on `bg-[#080F1E]` page background: the FCF fill is nearly the same dark value as the bar background. Fix: raise to `bg-blue-800` or `bg-[#1e40af]/60` so both segments read distinctly.

### P1 — % sub-rows create extreme density
8 percentage rows (Capex/Rev, EBIT Margin, D&A/Rev, NWC/Rev, NetDebt/Rev, NOPAT Margin, etc.) are always visible at the same weight as primary rows. These are secondary — they should be visually subordinate: lighter ink, tighter padding. This alone reduces perceived row count by ~40%.

### P2 — `text-slate-500` on dark background
Many sub-label cells use `text-slate-500` on `bg-[#080F1E]`. On a very dark navy, slate-500 (#64748B) yields approximately 3.8:1 — below the 4.5:1 body text requirement. Raise to `text-slate-400` minimum (4.8:1 on this bg).

### P2 — "Model" badge on growth cells clutters small cells
The `bg-blue-500/20 text-blue-400` "Model" sub-badge inside projected growth cells adds ~14px of height to an already-small cell. It's a good signal but should only appear on hover/focus, not permanently.

---

## Distill Recommendations

1. **Section headers**: Replace all `uppercase tracking-widest` labels with sentence-case. Use a left-border accent rule + normal-case `text-xs font-semibold text-slate-400` to group sections instead.
2. **% sub-rows**: Reduce `py-1.5` to `py-0.5`, drop from `text-[11px] text-slate-500` to `text-[11px] text-slate-600` (darker, still secondary), tighten padding to `pl-8` (more indented) to create clear primary/secondary hierarchy.
3. **Composition bar**: Change FCF segment from `bg-[#0F2A5E]` to `bg-blue-700` for clear visual separation.
4. **Inactive toggle text**: Change `text-slate-400` inside `bg-blue-600` buttons to `text-blue-100`.
5. **"Model" badge**: Show only on `:hover` / `:focus-within` of the cell.
6. **Header badge labels**: Remove uppercase from "Avg CAGR" and "Implied" chip labels.
